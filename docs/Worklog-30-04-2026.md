## Problems encountered

- Concurrent API processes bound to port 3000 caused `EADDRINUSE`, mixed process behavior, and inconsistent runtime debugging signal.
- Cold or delayed DB connectivity caused slow initial data readiness due to retry backoff.
- Post mobile-card rollout, action clicks appeared non-responsive until `DataTable` action event capture handling was tightened.

## Decisions made

- Keep desktop tables as primary view; use cards only on mobile within Distribution context.
- Standardize destructive operations on typed double-confirm prompts to reduce accidental data mutations.
- Prefer production fast-fail DB connect defaults and rely on process restart policy over long startup hangs.
