# SPHINCS Beta Release Notes

Version: `Beta Vx.y.z`
Date: `YYYY-MM-DD`
Phase: `Beta V1` or `Beta V2`

## What Changed

- Short bullet for the main shipped improvement
- Short bullet for the next most visible user-facing change
- Short bullet for any important fix or safety change

## What Testers Should Check

- Specific workflow 1
- Specific workflow 2
- Specific workflow 3

## Known Notes

- Mention any limitation, migration dependency, or expected rough edge
- Mention if a clean login/session is recommended after deploy

## If You Hit A Problem

- Use the in-app `Report Bug` flow where available
- Include what you were doing, what you expected, and what happened instead
- If the issue blocks access, include the module and screen you were on

Keep this short.
The goal is to help testers focus, not to bury them in a wall of text.
